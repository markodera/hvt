# Security Policy

## Supported Versions

The `main` branch is the actively supported open-source version.

Security fixes may also be backported to the latest release branch when a
maintainer decides the backport is low risk and necessary.

## Reporting a Vulnerability

Do not open a public GitHub issue for security vulnerabilities.

Use GitHub's private vulnerability reporting for this repository when it is
available. If private reporting is not enabled, contact a repository owner
directly before disclosing details publicly.

Include:

- a clear description of the issue
- the affected endpoints or code paths
- reproduction steps or a proof of concept
- impact assessment if known

We will aim to acknowledge valid reports within 5 business days and provide a
status update after triage.

## Disclosure

Please give maintainers reasonable time to investigate, mitigate, and release a
fix before public disclosure.
